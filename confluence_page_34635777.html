Purpose: Central index for all coding-agent session logs (Claude Code, pi, and other dev agents). Each child page documents one dev session — what was done, what was decided, what changed on disk, and how a future agent should resume. For conversational Rovo AI sessions, use the separate AI Session Logs index instead.

How logs get created

  • Automated: the PM daemon's log_agent_session MCP tool (scripts/pm/pm_daemon.py) publishes a page here at the end of a session. Titles follow Session Log: {agent} — {YYYYMMDD-HHMMSS}.

  • Manual: copy the child page Template - Agent Session Log and fill in the bracketed placeholders. Titles follow Session Log - YYYY-MM-DD - {branch} - {short description}.

Conventions

  • Always use UTC timestamps.

  • All agent session logs are child pages of this page.

  • Label every entry agent-session-log to enable CQL filtering: type = page AND label = "agent-session-log".

  • Never paste secrets, tokens, or .env contents into a log — record only branch, PR, Jira issue, and Confluence page references, and note redactions in the Sensitive Data section.

Configuration

The PM daemon publishes to this page when CONFLUENCE_SESSION_LOGS_PARENT_ID in the repo .env is set to this page's ID. Without it, logs fall back to the broader Project Documentation parent. See docs/PM_TOOLING.md.

Template health check

After 5 logged sessions, review the template: remove sections that stay empty, add sections for context that keeps going missing. Prior art: the 2026-05-19 session log (established the format) and the publish-path smoke test (dry-run).
